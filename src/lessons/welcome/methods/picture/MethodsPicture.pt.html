Desenho colorido

Neste exercício, vamos reproduzir o desenho geométrico que pode ver na guia "Objetivo".

O seu objetivo (aqui e em qualquer programa bem escrito) é escrever o código mais simples possível. Para isto, tem que decompor o seu trabalho em sub-passos e escrever métodos específicos para cada sub-passo.

Se observar cuidadosamente a figura para pintar, é constituída de quatro partes a descrever um tipo de V a usar uma cor diferente. Uma decomposição possível é escrever um método responsável por pintar um V de uma cor específica a partir da posição atual. O protótipo dela pode ser:

[!java|c]void [/!]fazerV([!java|c]Cor [/!]c[!scala]: Cor[/!])[!python]  # parâmetro c é do tipo Cor[/!]

O tipo de dados Cor descreve obviamente uma cor particular. O seu código provavelmente vai chamar fazerV com os seguintes argumentos (uma cor diferente para cada chamada):

  • [!java|scala|python]Cor.[/!]AMARELO
  • [!java|scala|python]Cor.[/!]VERMELHO
  • [!java|scala|python]Cor.[/!]AZUL
  • [!java|scala|python]Cor.[/!]VERDE

Em makeV(), deve usar o método setBrushColor() (pré-definido no buggle) para mudar a cor do pincel do buggle, assim como brushUp() e brushDown() para mudar a posição do pincel.

É recomendável escrever o código do makeV() de forma que deixe o buggle na posição ao próximo V.
